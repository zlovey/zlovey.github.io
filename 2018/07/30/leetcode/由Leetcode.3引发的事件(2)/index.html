最长公共子序列（Longest Common Subsequence,LCS）

问题描述：例子：有两个母串

  • cnblogs
  • belong

比如序列bo, bg, lg在母串cnblogs与belong中都出现过并且出现顺序与母串保持一致，我们将其称为公共子序列。最长公共子序列，顾名思义，是指在所有的子序列中最长的那一个。子串是要求更严格的一种子序列，要求在母串中连续地出现。在上述例子的中，最长公共子序列为blog（cnblogs,belong），最长公共子串为lo（cnblogs, belong）。

问题分析：

1.暴力求解O（2^n)

列举两个母串中较短那个的所有2^n个子序列，对每一个子序列判断其是否是另一母串的子序列。

字符串查找

问题描述：KMP算法要解决的问题就是在字符串（也叫主串）中的模式（pattern）定位问题。说简单点就是我们平时常说的关键字搜索。模式串就是关键字（接下来称它为P），如果它在一个主串（接下来称为T）中出现，就返回它的具体位置，否则返回-1）。

1.暴力匹配

通过i,j记录当前模式串和主串的匹配位置，当匹配成功时，记录位置向后移动；当匹配失败的时候，使模式串的标志位回到起始位置，主串的标志位回到当前起始匹配的下一个位置。

我们会把T[0] 跟 P[0] 匹配，如果相同则匹配下一个字符，直到出现不相同的情况，此时我们会丢弃前面的匹配信息，然后把T[1] 跟 P [0]匹配，循环进行，直到主串结束，或者出现匹配成功的情况。这种丢弃前面的匹配信息的方法，极大地降低了匹配效率。

每次匹配失败的时候，将主串在上次匹配的基础上仅向后移动一位，而再对模式串进行匹配，在坏的情况下，时间复杂度可以达到O(n*m)。

然而在这过程中，有很多重复匹配的数据，在上例中，第二次查找的时候，主串又回到第二位B的位置，而模式串回到第一位A的位置。如果是人为来寻找的话，肯定不会再把i移动回第1位，因为主串匹配失败的位置前面除了第一个A之外再也没有A了，我们为什么能知道主串前面只有一个A？因为我们已经知道前面三个字符都是匹配的！（这很重要）。移动过去肯定也是不匹配的！

如何使i不动，而只需要移动j即可，如下图所示，不用再重复进行主串的遍历，减少了多余的时间浪费。这就是大佬们提出的KMP算法。

2.KMP

在KMP算法中，对于每一个模式串我们会事先计算出模式串的内部匹配信息，在匹配失败时最大的移动模式串，以减少匹配次数。

由上所知，KMP的问题在于当主串与模式串不匹配时，应该将模式串移到什么位置，我们将每一位不匹配时应该移动到的位置用next数组进行存储。就可以将之前的算法修改如下：

因此我们可以看出，模式串应该移动到的位置k有相同的特性，即模式串最前面的k个字符和匹配失败的模式串j位置之前的k个字符是一样的。 对模式串来说，next 数组各值的含义代表当前字符之前的字符串中，有多大长度的相同前缀后缀。例如如果next [j] = k，代表j之前的字符串（一定不包括j）中有最大长度为k 的相同前缀后缀。

由于每个位置都可能不匹配，故需要求得next数组，存储每一次不匹配时应该将模式串移动到哪个位置，而要得到next数组，又需要寻找模式串中前缀后缀最长公共元素的长度。

例如：当ABCDA时，出现了公共 的前后缀A，其长度为1；而当ABCDAB时，有公共前后缀AB，其长度为2，当ABCDABD时，没有公共的前后缀。

模式串 A B C D A B D
前后缀最大公共元素长度 0 0 0 0 1 2 0
模式串 A B C D A B D
next数组 -1 0 0 0 0 1 2

next数组是在该字符之前的串的前后缀最大公共元素长度，就相当于将第一个表的整体结果向后移动并将第一个数值补齐为-1。
